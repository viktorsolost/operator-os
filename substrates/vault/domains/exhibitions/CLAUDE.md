# Exhibition Domain Context

This folder holds durable exhibition-specific context that applies across projects and project types when the work is fundamentally exhibition-related.

Use this folder for:
- exhibition vocabulary
- recurring exhibition operating patterns
- common risks and dependencies
- durable assumptions that repeat across exhibition work

Do not put project-specific live facts here.
Do not put runtime approvals, task state, or session history here.

Routing:
- Read `glossary.md` when the issue is terminology, interpretation, or role meaning.
- Read `rules.md` when the issue is process, risk, or practical domain behavior.
- If the issue is about a recurring project structure, also check `../../project-types/exhibition/`.
- If the issue is about one specific project, go to `../../intake/` or `../../projects/`.
