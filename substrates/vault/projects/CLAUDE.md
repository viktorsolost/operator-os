# Active Projects

This folder is for approved active projects.

A folder in `projects/` means:
- project identity is accepted
- setup direction is sufficiently resolved
- the project is an active execution surface

Use this folder for:
- project-local truth
- stakeholders
- constraints
- durable decisions
- references
- optional task-level deeper context
- session logs (`session-log.md`) — detailed record of what was done per session, decisions made, pending items

Standard project folder files:
- `summary.md` — project state and key facts
- `references.md` — links, contacts, file pointers
- `decisions.md` — logged decisions with status
- `open-questions.md` — unresolved questions
- `session-log.md` — detailed session work log (append new sessions at the top)

Routing:
- If the issue is still unresolved intake/setup ambiguity, check `../intake/`
- If the issue is about Viktor-wide guidance, check `../operator/CLAUDE.md`
- If the issue is about ambiguity handling, check `../operator/clarification-protocol.md`
- If the issue needs durable cross-session truth, check `../memory.md`
- If the issue needs likely relevant nearby-session context, check `../recent-context.md`
- If the issue is about recurring project-type structure, check `../project-types/`
- If the issue is domain-specific, check `../domains/`
