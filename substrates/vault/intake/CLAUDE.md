# Intake Projects

This folder is for newly identified projects that are not yet approved active projects.

A folder in `intake/` means:
- the system believes a likely real project exists
- identity may still be ambiguous
- setup decisions may still be unresolved
- this is not yet an approved active project

Use this folder for:
- early identity and framing
- open setup questions
- clarification and routing
- approval readiness

Do not treat intake projects as active execution surfaces by default.

Routing:
- If the issue is about operator-wide preferences, check `../operator/identity.md` and `../operator/working-style.md`
- If the issue is about ambiguity handling, check `../operator/clarification-protocol.md`
- If the issue needs durable cross-session truth, check `../memory.md`
- If the issue needs likely relevant nearby-session context, check `../recent-context.md`
- If the issue is about recurring project-type structure, check `../project-types/`
- If the issue is domain-specific, check `../domains/`
- If the issue is whether to create or reuse SOP/calendar structure, keep that reasoning here until resolved
- If the project is approved and active, promote it to `../projects/`

- project-type confirmation
- asset-by-asset setup decisions

- strategy approval guidance
- setup approval guidance
- promotion readiness

Do not treat intake as a single yes/no gate.

- milestone structure
- SOP
- production calendar
- drive setup
