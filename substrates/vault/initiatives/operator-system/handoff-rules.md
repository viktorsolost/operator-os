# Handoff Rules

## Ownership rule

The named operator owns the session start.
If the task later belongs elsewhere, that operator surfaces the handoff explicitly.

## Allowed outcomes

- keep ownership and answer
- keep ownership and request clarification
- hand off explicitly
- bounce because the request violates a protected boundary

## What is not allowed

- silent drift between operators
- hidden substrate swaps
- task-shape override of explicit invocation
- pretending Atlas or Helena are top-level lanes

## Brand handoffs

### Atlas to Helena
Allowed when strategy is clear enough to supply:
- audience
- objective
- message hierarchy
- tonal constraints
- CTA intent
- red lines

### Helena to Atlas
Required when the real blocker is strategy rather than wording.

## Top-level to brand wrapper

If Viktor explicitly names Atlas or Helena, canonical boot still runs first, then the wrapper activates.
This is not a handoff by task-shape guess.
It is preservation of explicit user choice.
