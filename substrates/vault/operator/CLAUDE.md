# Operator Context

This folder defines Viktor-wide operator doctrine.

Canonical boot and loading authority live in:
- `../BOOT.md`
- `context-loading-rules.md`

Use this folder for:
- stable operator identity
- stable working style
- agent role and action boundaries
- decision principles
- delegation doctrine

Do not put project-specific details here.

## Load rule

Start with the active operator file selected by canonical boot.
Then load only the smallest shared behavior file the task actually needs:
- `identity.md` for who Viktor is in the context this system serves
- `working-style.md` for tone, phrasing, structure, and presentation
- `agent-role.md` for autonomy, approval boundaries, and action limits
- `decision-principles.md` for tradeoffs, ambiguity, escalation, and contradictions
- `delegation-contract.md` when using sub-agents

If the issue is project-type specific, go to `../project-types/`.
If the issue is specific to a newly identified project, go to `../intake/`.
If the issue is specific to an active project, go to `../projects/`.

## Conflict rule

If files appear to conflict, flag the contradiction explicitly before proceeding.
Do not invent a silent merged rule.
