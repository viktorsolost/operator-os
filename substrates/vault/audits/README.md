# Audits

This folder holds audit artifacts, review outputs, and historical validation material.

Use this folder for:
- audits of the operator system
- audits of routing or structure
- review snapshots
- validation packs
- historical evidence from audits

Do not use this folder for:
- live canonical rules
- active project memory
- runtime state

Rule:
If an audit produces a hard live rule, that rule must be distilled upward into the canonical files.
The audit itself stays here as supporting history.
