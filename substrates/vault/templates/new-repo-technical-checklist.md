# New Repo Technical Checklist

Use this during the first Anton/Jonah pass in a new codebase.

## Inspect

- Repo root structure reviewed
- Main app/runtime entry points identified
- Build command identified
- Test command identified
- Main package/runtime files identified
- State, storage, or database locations identified
- Existing guidance files identified

## Create

- Local `CLAUDE.md`
- `docs/engineering/README.md`
- `docs/engineering/sessions/`
- `docs/engineering/areas/`
- First onboarding session note

## Clarify

- Canonical source-of-truth boundaries documented
- Primary architecture areas documented
- Known stale docs flagged
- Known risks or debt flagged
- Commands for run/build/test captured

## Handoff

- Anton has defined the implementation shape
- Jonah has verified the proposed shape against the repo
- Future engineering work can route through local docs without re-discovery
