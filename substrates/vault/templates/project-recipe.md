# Project Recipe

Use this file as the simple rule for what a project folder should contain.

Keep this small.

Do not add files unless they help real work.

## Core files for most active projects

- `summary.md` — what the project is, its current state, and the key facts
- `references.md` — links, people, documents, and source pointers
- `decisions.md` — durable project decisions
- `open-questions.md` — unresolved questions that affect execution
- `session-log.md` — chronological record of what happened and what changed

## Optional files and folders

Add extra files only when they clearly help.

Examples:
- `calls/` — call notes for projects with many meetings
- `drafts/` — working drafts if they need to stay project-local
- `assets/` — small project-local supporting files
- topic-specific notes when the project genuinely needs them

## Rules

- Start from the template files in `templates/active-project/`
- Keep durable meaning in the vault, not raw runtime dumps
- Put detailed project history in `session-log.md`, not in global memory
- Do not create fancy substructure unless the project actually needs it
- Prefer adding one useful file over inventing a deep folder tree

## Intake projects

Projects in `intake/` use the intake template instead.

They should stay lighter until the project is accepted into active execution.
