# Product Framing

## Framing

Sera is not just a dashboard.

Sera is a personal execution environment that combines:

- context
- memory
- agent help
- controlled actions

## Layers

1. Upstream truth

- Basecamp
- Google Sheets
- Drive
- Calendar
- email and other live systems

2. App truth

- local runtime state
- project store
- task state
- derived execution data

3. Meaning layer

- markdown notes
- decisions
- reasoning
- open questions
- durable project/app memory

## Operating Model

- Sera is the system
- Claudia is the main agent
- more agents can exist later with separate roles and rules

## Current Direction

Near-term work is still about tightening architecture and creating durable internal reasoning structure, not yet broad rollout.
