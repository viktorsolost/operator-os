# Setup Model

## Repo vs Vault

### Repo

Use the repo for:

- code
- tests
- runtime logic
- APIs
- UI
- implementation notes that are code-adjacent

### Vault

Use the vault for:

- product reasoning
- brainstorming
- decisions
- app rollout thinking
- agent definitions
- pilot-user thinking
- durable non-code memory
