Four operators looked at the same idea. They all saw the same product trying to get out.

This document synthesizes all four proposals into one direction: half vision, half reality. What Memento wants to be, and what we can ship by stripping VIK OS to its honest core.

Universal Agreement

Four different minds. One product.

Anton
"I maintain truth. Memento handles the rest." The product is a context engine with an execution loop, not an app with AI features.
Claudia
"Every question the system asks the user is a failure of context, not a failure of intelligence." Trust is earned through accuracy, never toggled in settings.
Jonah
"Done is the product." The only measure: how much got done today that the owner didn't have to do? 91% of VIK OS is scaffolding for a weak engine.
Lev
"The fundamental error of every AI product is that they are apps." Memento is a presence. The core intelligence is not knowledge. It's knowing when something matters.

Where all four converge: Memento is not a dashboard. It's not a task manager. It's not an AI chatbot. It's a continuous loop that reads context, reasons about what matters, acts through existing channels, and earns trust over time. Conversation is the primary interface. UI exists only where conversation fails.

The Product

Context in. Done out.

Three systems running continuously. Not on a schedule. Not waiting for the user. Running.

Observer: Reads email, calendar, messages, documents. Builds a living world model of the person's reality. Projects, people, commitments, patterns, rhythms. Never asks for input. Learns from signals already being produced.

Reasoner: Given the world model, what should happen next? Not what the person asked for. What should happen based on everything that's true right now. Connects events to threads to deadlines to relationships. This is where the LLM earns its keep. Not answering questions. Making judgment calls.

Executor: Does the work. Sends emails, moves meetings, follows up, pays invoices, prepares briefs. Through the channels the person already uses. Trust earned over time through demonstrated accuracy.

From Claudia

The Trust Ladder

Observe. Reflect. Anticipate. Propose. Act. Each rung earned, not granted. The system demonstrates competence until the user says "just handle it." This is the product's growth mechanic.

From Jonah

Completion Levels

Reactive (respond to what happened). Proactive (initiate what should). Preventive (stop what's coming). Strategic (maintain life trajectory). Most AI stops at level 1.

From Lev

Temporal Intelligence

The same information is worthless at 2am and life-changing at 9:03am walking into a meeting. The core intelligence is not what. It's when. Match the intervention to the state of the person.

From Anton

Five Primitives

People. Projects. Commitments. Context. Actions. Everything else is derivative. A meeting is a commitment. A note is context. A task is an action. Keep the model small and durable.

The Two Paths

Both are the same product at different stages

Stripped VIK OS

Ship now

Take the existing pipeline, the existing data, the existing integrations. Strip the UI to Today + Terminal + Approvals. Make the agent terminal the primary interaction. This is Memento v0 running on proven infrastructure. The context engine is still pipeline-shaped, but the interface is already honest.

Memento

Build toward

Replace the pipeline with a continuous observer. Replace the store with a living world model. Replace the terminal with natural conversation. Replace approval pages with trust earned over time. Same contract, new architecture. This is where the product wants to go.

They're not competing visions. Stripped VIK OS is the honest v0. Memento is the real product. The v0 proves the core loop works. Then the architecture evolves underneath while the contract stays the same: you live your life, the work gets done.

Path Forward

From stripped VIK OS to Memento

Phase 1: Strip
Cut VIK OS to three surfaces. Today feed, agent terminal, approvals. Remove all other pages. Pipeline runs headless. Existing data and integrations stay. Validate that context-in, done-out works with minimal UI.
Phase 2: Converse
Make conversation the primary interface. The agent terminal grows into natural dialogue. Voice input and output. The today feed becomes secondary. The system starts proposing actions, not just surfacing information.
Phase 3: Act
Memento starts doing work. Sending emails, moving meetings, following up. Trust ladder begins. Low-stakes actions are silent. High-stakes actions ask. The user starts saying "just handle it."
Phase 4: Learn
The world model deepens. Patterns, rhythms, relationship dynamics, energy states. The system knows when to surface information, not just what. Temporal intelligence from Lev's proposal becomes real.
Phase 5: Generalize
Memento works for anyone. Remove Viktor-specific assumptions. Universal onboarding: connect email and calendar. The world model builds itself. Subscription product ready.
Phase 6: Disappear
The interface dissolves. The app opens less. Voice handles more. Actions are mostly silent. The metric is relief, not engagement. The product measures itself by how little it's needed visibly.

Strong systems need less interface.
The measure is what got done that the owner didn't have to do.
